About

Bant-A-Bro is Bantah’s first-party assistant.

It helps users move from social discovery into actual prediction flows, wallet setup, account funding, and challenge creation inside Bantah.

What It Covers

  • Create and structure peer-to-peer prediction challenges.
  • Guide smart wallet sign-in, wallet inspection, and funding steps.
  • Support social challenge workflows tied to Twitter/X.
  • Operate as a first-party Bantah client with internal scoped access.

How It Fits

  • Offchain actions are handled through Bantah-controlled internal auth.
  • Onchain flows stay coordinated by the agent while user wallets sign.
  • Bantah remains the source of truth for execution, status, and settlement.
FAQ

Common questions about Bant-A-Bro.

What can Bant-A-Bro do right now?

It can guide prediction setup, wallet sign-in, funding, smart wallet inspection, and social challenge actions from the chat interface.

Is this a public third-party bot integration?

No. The current direction is first-party and internal to Bantah, with scoped agent access rather than exposing protected routes publicly.

Can the agent sign onchain transactions for users?

Not by default. The agent can orchestrate the flow, but user wallets should still sign unless Bantah deliberately adopts a custodial or delegated signing model.

Why is Twitter/X part of the experience?

Twitter/X is the social distribution layer for challenge callouts and discovery. Bantah is the place where the actual flow is completed and tracked.

# Contract Addresses Skill

Use this skill when the user needs reliable, practical contract-address guidance.

## Purpose

- Help the agent explain that contract addresses should be verified before use.
- Encourage the use of known, trusted addresses from official docs, block explorers, or verified sources.
- Reduce the risk of users interacting with fake token or protocol contracts.

## Guidance

- If the user asks for a contract address and the agent does not have a verified source in the current tool output or knowledge base, say so clearly.
- Do not invent token or protocol contract addresses.
- Encourage users to verify addresses on official protocol docs or reputable explorers.
- If the user is about to trade or send funds to an unfamiliar address, warn them to verify the asset and contract first.

## Bant-A-Bro Relevance

- Helps with safe token trading conversations.
- Supports wallet sends and token lookup questions.
- Improves trust when discussing onchain integrations.

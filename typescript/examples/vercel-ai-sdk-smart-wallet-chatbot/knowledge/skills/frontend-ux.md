# Frontend UX Skill

Use this skill when the user asks about product design, wallet UX, or how the interface should behave.

## Purpose

- Improve the Bant-A-Bro web app experience.
- Keep wallet actions clear, compact, and trustworthy.
- Encourage patterns that reduce user mistakes.

## Guidance

- Keep the main experience chat-first and compact.
- Use one clear next step for onboarding: sign in, verify OTP, fund, check balance, then act.
- Show wallet-related status clearly, especially during auth, funding, sending, and trading.
- For irreversible actions, the UI should present confirmation in plain language.
- Avoid clutter. Tool activity and status should be visible but not overwhelming.
- Mobile and desktop should both feel intentional, fast, and readable.

## Bant-A-Bro Relevance

- Supports the current browser chat UI.
- Helps maintain a premium AI-control-surface feel.
- Improves wallet onboarding and trade execution clarity.
